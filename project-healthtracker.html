The Challenge

Most habit trackers fail for the same reason: they're designed to log behaviour, not change it. The data goes in, nothing meaningful comes out. The product becomes a record of failure rather than a scaffold for progress.

This project set out to design a habit tracker that understood the psychology of behaviour change - one that used streaks, reflection prompts, and pattern recognition to actually move the needle, not just capture it.

Research and Discovery

The project began with a structured research phase - user interviews, competitive audit, and persona development. Three distinct user archetypes emerged: the Optimiser (high motivation, data-hungry), the Rebuilder (recovering from burnout, needs gentleness), and the Maintainer (consistent behaviour, light-touch tracking).

The design had to serve all three without defaulting to the most vocal one - which in healthtech is almost always the Optimiser, at the expense of everyone else.

Designing for the Rebuilder forced better decisions for everyone. Removing friction is always right; removing accountability is not.

What I Built

The final product is a React single-page application with local state management, designed for daily interaction and built with accessibility front-of-mind throughout.

  • Daily check-in flow with adaptive prompt logic based on streak status
  • Habit creation wizard with categorisation (health, productivity, wellbeing)
  • Weekly review screen surfacing pattern analysis and trend data
  • Streak visualisation with deliberate "compassionate restart" design to reduce shame spirals
  • Dark mode and high contrast mode support throughout
  • Progressive web app (PWA) support for mobile home screen installation

Usability Testing Process

Three rounds of moderated usability testing were run before launch, each targeting a different user archetype. Key findings from each round fed directly into the next design iteration - nothing was carried forward without evidence of improved task completion or reduced friction.

The most impactful change to emerge from testing: the original "missed streak" notification was actively demotivating. Users reported feeling punished rather than prompted. Replacing it with a simple, low-tone "pick up where you left off" message increased seven-day retention in the test cohort by a meaningful margin.
